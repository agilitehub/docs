## Test using /getByProfilekey

Go to the [Agilit-e API Reference](/reference) page.

1. Navigate to the Keywords Execution APIs category under the Agilit-e Keywords section, followed by clicking on the `/getByProfileKey` menu item.

![882](/images/docs/a5166b6-Screen_Shot_2018-10-08_at_12.40.48.png "Screen Shot 2018-10-08 at 12.40.48.png")

2. Click on the user icon on the top right of the API logic and populate your generated Agilit-e token into the `api-key` field.

<Warning>
  **Note:** You will need to create an API Key, as mentioned earlier in the [API Keys Module](/docs/api-keys-overview) section of the documentation. Once the API Key is provided for one API, it will exist for the rest of the APIs.
</Warning>

![700](/images/docs/56d4f12-api-key-authentication.png "api-key-authentication.png")

3. Under the Headers section, provide the `profile-key`.

![2040](/images/docs/fa8ccda-Keywords-GetByProfileKey-Headers.png "Keywords-GetByProfileKey-Headers.png")

4. Click on the `Try It` button to execute the API.

* The API response will contain the `Leave Type` keywords that were configured in the Keywords profile.

![776](/images/docs/76692e8-Keywords-GetByProfileKey.png "Keywords-GetByProfileKey.png")
